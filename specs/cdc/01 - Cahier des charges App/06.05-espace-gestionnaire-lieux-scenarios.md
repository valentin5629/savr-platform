# Scénarios de test — §06.05 Espace client gestionnaire de lieux

**Source CDC** : §06.05 + §05 règles R_taux_recyclage, R_dechets_labo_estimes, R_lieux_admin_only_fields + §04 (`organisations_lieux`, `coefficients_perte_labo`, `f_benchmark_kg_pax_zd`, `v_collectes_gestionnaire_lieux`) + §09 (matrices users/evenements/factures/packs/tarifs_negocie/shared.fichiers)
**Généré le** : 2026-06-07
**Statut** : À implémenter par Claude Code

> **Instructions Claude Code** : ces scénarios sont la source de vérité pour les tests du module §06.05.
> Pour chaque scénario :
> - Couche `db` → écrire un test pgTAP dans `supabase/tests/`
> - Couche `api` → écrire un test Vitest dans `packages/plateforme/tests/api/`
> - Couche `ui` → écrire un test Playwright dans `packages/plateforme/tests/e2e/`
> Les tests P1-critique sont bloquants CI. Les tests P2 et P3 sont non-bloquants V1.
> Ne pas démarrer le développement du module sans avoir écrit les tests P1 d'abord.

**Décisions Val 2026-06-07 (6 floues tranchées, propagées §06.05 + §09)** :
- **F1** : toggle « email à chaque nouvelle collecte programmée sur un de mes lieux » **supprimé V1** (aucun template ne l'implémentait — pattern sobriété 2026-05-30). Réintroduction V1.1 avec template dédié si demande terrain.
- **F2** : **statut consolidé événement** défini — `Annulé` = toutes collectes `annulee` ; `Terminé` = toutes collectes terminales (`realisee`/`cloturee`/`annulee`) dont ≥ 1 `realisee` ou `cloturee` ; `En cours` = sinon (≥ 1 collecte non terminale). Mapping affichage statut collecte : `programmee`/`validee` → « Programmée », `realisee`/`cloturee` → « Réalisée », `annulee` → « Annulée ».
- **F3** : **brouillons tiers exclus** de la visibilité gestionnaire — prédicat SELECT `evenements` : `(date_evenement IS NOT NULL OR organisation_id = self)`.
- **F4** : UPDATE `evenements` gestionnaire **gaté par `f_collecte_editable(evenements.id)`** (aligné workflow traiteur).
- **F5 BLOQUANT** : matrice `users` §09 — gestionnaire_lieux aligné traiteur_manager : INSERT + UPDATE `organisation_id = self` (invitation + désactivation). Garde UI : pas d'auto-désactivation.
- **F6 BLOQUANT** : matrice `factures` §09 — gestionnaire_lieux SELECT `organisation_id = self` (ses propres factures uniquement ; factures des traiteurs toujours invisibles). Miroir `shared.fichiers` (scope strict = RLS table factures). pgTAP `fichiers_facture_gestionnaire_denied` scindé en 2.
