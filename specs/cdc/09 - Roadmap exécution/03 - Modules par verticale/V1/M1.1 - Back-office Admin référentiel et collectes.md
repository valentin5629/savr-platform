# Brief Claude Code — Module 1.1 — Back-office Admin : référentiel & gestion collectes

> Révisé 2026-06-11 (conventions `12 - Conventions exécution`). Module XL → **3 sous-lots, 1 /goal et 1 checkpoint par sous-lot**.

## Objectif
Construire le back-office Admin Savr : CRUD référentiel (orgas, users, lieux, événements) + liste/gestion des collectes avec statuts et filtres + dashboard Admin de base. C'est la colonne vertébrale opérationnelle qui permet à Savr de gérer les données manuellement pendant les tests.

## Contexte (lu depuis CLAUDE.md)

## Sous-lots (ordre imposé)
- **M1.1a — Référentiel orgas & users** : CRUD organisations (création/modif/désactivation), utilisateurs (invitation/rôle/impersonation), endpoints + RLS staff. ~450k.
- **M1.1b — Lieux, événements & collectes** : CRUD lieux (référentiel + validation manuelle), événements + collectes (saisie + modif), vue liste collectes (statuts, filtres, actions rapides). ~550k.
- **M1.1c — Dashboard Admin de base** : collectes programmées / en cours / incidents / acceptance TMS ; triggers référentiel si requis (audit via 0.10). ~400k.
