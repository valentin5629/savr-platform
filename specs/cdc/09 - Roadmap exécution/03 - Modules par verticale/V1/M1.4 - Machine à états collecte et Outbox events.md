# Brief Claude Code — Module 1.4 — Machine à états collecte + Outbox events

> **Régénéré 2026-06-11** — intègre la revue adversariale concurrence (R1 : row lock agrégat avant INSERT outbox) et le modèle d'émission validé frère (RPC dispatch, pas de trigger brut). Remplace le brief du 2026-06-08.

## Objectif
Implémenter la machine à états unifiée de la collecte (`programmee → validee → en_cours → realisee → cloturee`) et le **transactional outbox côté émission** : toute mutation métier émettant un event (E1 `collecte.creee`, E2 `collecte.modifiee`, E3 `collecte.annulee`, E5 `lieu.champ_critique_modifie`) écrit une ligne `outbox_events` **dans la même transaction** (garde-fou 4).

## Pièges connus / décisions à ne PAS rouvrir
- Transactional outbox **non négociable** (garde-fou 4) : un test pgTAP par mutation E1/E2/E3/E5 + test d'atomicité rollback (`outbox_par_mutation.test.sql`, auto-activé, déjà câblé au harnais).
- Émission = RPC/action métier, jamais trigger brut sur UPDATE (décision frère).
- `seq` = bigserial + le row lock R1 ; ne pas « optimiser » en réordonnant lock/insert.
- Annulation poussée TMS → E3 systématique tous acteurs (exigence Val, F5 06.04).
